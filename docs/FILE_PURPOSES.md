# Назначение файлов и папок — Neetrino

**Назначение документа:** ориентир для разработчика: зачем нужна каждая **логическая область** репозитория и основные файлы в ней.  
**Ограничение:** не перечислены тысячи файлов зависимостей (`node_modules/`), сотни экспортов из Figma в `public/figma-assets/`, и каждый SVG/PNG по отдельности — для них достаточно описания **папки**.

---

## Корень репозитория

| Файл / папка         | Зачем нужен                                                                      |
| -------------------- | -------------------------------------------------------------------------------- |
| `package.json`       | Имя проекта, скрипты (`dev`, `build`, `lint`, тесты, ассеты), зависимости.       |
| `pnpm-lock.yaml`     | Зафиксированные версии пакетов для воспроизводимых сборок (pnpm).                |
| `tsconfig.json`      | Настройки TypeScript (strict, пути `@/*` и т.д.).                                |
| `next.config.ts`     | Конфигурация Next.js (бандл, изображения, эксперименты — по мере необходимости). |
| `next-env.d.ts`      | Автогенерация типов Next.js; не править вручную.                                 |
| `vercel.json`        | Правила деплоя на Vercel (редиректы, заголовки и т.п., если заданы).             |
| `components.json`    | Конфиг генератора shadcn/ui (алиасы путей, стиль компонентов).                   |
| `postcss.config.mjs` | Подключение PostCSS (в т.ч. Tailwind v4).                                        |
| `README.md`          | Входная точка для человека: как запустить проект локально.                       |
| `AGENTS.md`          | Подсказки для AI-агентов (например особенности Next.js в проекте).               |
| `CLAUDE.md`          | Ссылка/напоминание для Claude — обычно `@AGENTS.md`.                             |
| `.gitignore`         | Что не коммитить (артефакты сборки, env, IDE).                                   |
| `.env*`              | Локальные секреты и переменные окружения (не коммитить реальные значения).       |

## Как поддерживать документ в актуальном виде

- При добавлении **нового маршрута** в `app/` — строка в таблице `app/`.
- При новой **фиче-папке** в `components/` — краткий подраздел или строка в соответствующей таблице.
- При новом **скрипте** в `scripts/` — одна строка в таблице скриптов.

Если нужен полный машинно-читаемый список файлов, используйте поиск по репозиторию или `git ls-files` — этот документ намеренно остаётся **человекочитаемой картой**, а не дублем файловой системы.
